# 单词熟练度算法

本文档说明当前单词熟练度分数、等级和抽词优先级的计算方式。实现位置：`apps/web/src/lib/masteryCalculator.ts`。

## 目标

熟练度分数用于表示用户对某个单词的长期掌握程度；抽词优先级用于决定练习时更应该出现哪些单词。两者语义不同：熟练度不直接因为几天没练而下降，但长期未练会显著提高抽词优先级。

## 数据字段

每个单词会记录以下练习数据：

- `correctCount`：正确输入次数。
- `totalAttempts`：总尝试次数，包含正确输入、看提示或明显拼错。
- `inputTimes`：最近若干次正确输入耗时，单位为秒。
- `lastPracticedAt`：最近一次练习时间。
- `correctPracticeDates`：发生过正确输入的自然日列表，格式为 `YYYY-MM-DD`。

旧数据没有 `correctPracticeDates` 时，会在读取时根据 `lastPracticedAt` 和历史正确次数补一个日期，避免旧用户数据全部从零开始。

## 熟练度分数

总分范围为 0 到 100，由四个子分数组成：

| 因素 | 权重 | 说明 |
| --- | --- | --- |
| 正确率 | 50% | 使用平滑后的 `correctCount / totalAttempts`，降低小样本波动 |
| 输入速度 | 15% | 使用最近 5 次正确输入耗时，与按单词长度估算的期望时间比较 |
| 稳定性 | 20% | 使用最近 10 次输入耗时的变异系数，越稳定分数越高 |
| 跨天复习 | 15% | 根据有正确输入记录的自然日数量计算，最多按 3 天给满分 |

相比旧算法，输入速度权重降低，正确率和跨天复习的重要性提高。这样可以减少“打字快但记忆不稳”带来的高估。

## 等级门槛

分数对应等级如下：

| 等级 | 分数范围 |
| --- | --- |
| new | 0-19 |
| learning | 20-39 |
| familiar | 40-59 |
| proficient | 60-79 |
| mastered | 80-100 |

此外还会按练习次数和跨天正确记录设置等级上限：

- 少于 3 次尝试：最高 `learning`。
- 少于 5 次尝试，或少于 2 个正确复习日：最高 `familiar`。
- 少于 8 次尝试，或少于 3 个正确复习日：最高 `proficient`。

因此，单词不能只靠同一天连续输入几次就进入 `mastered`，必须经过多次且跨天的正确复习。

## 抽词优先级

抽词优先级越高，练习时越容易被选中。它由三部分组成：

- 基础优先级：熟练度越低，优先级越高；即使熟练度是 100，也保留最低基础优先级，避免永远不复习。
- 时间优先级：距离上次练习越久，优先级越高；7 天以上会明显提高，确保定期复习。
- 练习次数优先级：新词和低练习次数单词会被额外提高。

## 调参原则

- 如果用户觉得熟词出现太频繁，优先降低 7 天以上的时间优先级倍率。
- 如果用户觉得单词太容易变成 `mastered`，优先提高 mastered 的尝试次数或正确复习日门槛。
- 如果用户觉得打字速度影响过大，继续降低速度权重并把权重分配给正确率或跨天复习。
- 如果未来要接近 Anki/SM-2，可以在当前字段基础上继续加入动态复习间隔和 ease factor。
